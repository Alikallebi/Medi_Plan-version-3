# Guide Admin Workflow (Super Admin / Admin GTA)

## Périmètre
Le rôle admin couvre la supervision, les actions correctives et la traçabilité complète du workflow.

## 1) Configuration des workflows
- Vérifier les étapes de validation actives par service.
- Contrôler la présence d'une validation finale Super Admin si requise.
- Versionner les configurations importantes.

## 2) Dashboard et supervision
Depuis le dashboard admin :
- Suivre les KPI (en attente, retard, taux d'approbation).
- Identifier les plannings bloqués.
- Analyser la performance des validateurs.

## 3) Actions admin
- **Relance** : notifier le validateur en attente.
- **Réaffectation** : transférer la validation à un autre validateur.
- **Validation d'office** : action exceptionnelle à tracer.

## 4) Audit et traçabilité
- Utiliser l'**Audit Trail** global pour investiguer.
- Filtrer par période, utilisateur, type d'événement, planning.
- Exporter les données (`pdf`, `excel`, `csv`, `json`) selon le besoin.

## Recommandations
- Utiliser la validation d'office uniquement en dernier recours.
- Documenter les décisions sensibles via commentaires.
- Réaliser une revue hebdomadaire des plannings en retard.
